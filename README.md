# 🎵 SongVault — Premium Music Library

> A full-stack **Song Management Web Application** built with **Angular 21** (Frontend) and **Spring Boot** (Backend). Perform full CRUD operations on your personal music collection through a beautiful, modern UI.

## ✨ Features

- 🎵 **View all songs** in a responsive glassmorphism card grid
- ➕ **Add new tracks** via a clean, validated form
- ✏️ **Edit existing tracks** with pre-populated data
- 🗑️ **Delete songs** with an inline confirmation prompt (no accidental deletes!)
- 📊 **Live stats** — shows total tracks and unique artists count
- ✅ **Success & Error toasts** with smooth slide-in animations
- 📱 **Fully responsive** — works on mobile, tablet, and desktop
- ⚡ **Real-time backend sync** — all changes persist via REST API

## 🗂️ Data Model

```typescript
export interface Song {
  id?:          number;        // Auto-generated by backend
  title:        string;        // Song title
  artist:       string;        // Artist / band name
  genre:        string;        // Music genre (e.g. Pop, R&B)
  duration:     number;        // Duration in seconds
  releaseDate:  string | Date; // ISO date string (YYYY-MM-DD)
}
```

## 🔌 API Endpoints

The frontend communicates with the **Spring Boot backend** running on `http://localhost:8080`.

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/songs/allsongs` | Fetch all songs |
| `GET` | `/songs/{id}` | Fetch a single song by ID |
| `POST` | `/songs/add` | Create a new song |
| `PUT` | `/songs/update/{id}` | Update an existing song |
| `DELETE` | `/songs/{id}` | Delete a song |

## 🚀 Getting Started

### Prerequisites

- **Node.js** ≥ 18.x → [Download](https://nodejs.org/)
- **npm** ≥ 10.x (comes with Node)
- **Angular CLI** ≥ 21.x
- **Java** ≥ 17 (for the backend)
- **Maven** (for the backend)

---

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/song-vault.git
cd song-vault
```

---

### 2. Start the Backend (Spring Boot)

```bash
# From the backend project root
cd backend
mvn spring-boot:run
```

> The backend API will start at **http://localhost:8080**

---

### 3. Start the Frontend (Angular)

```bash
cd song-frontend
npm install
npm start
```

> The app will open at **http://localhost:4200**

## 🌐 Application Routes

| Route | Component | Description |
|---|---|---|
| `/` | Redirect | Redirects to `/songs` |

| `/songs/edit/:id` | `SongFormComponent` | Form to edit an existing track |

## 🎨 UI Design System

The app uses a custom CSS design system defined in `styles.css`:

- **Color Palette**: Deep Violet (`hsl(263,70%,8%)`) + Rose (`hsl(340,90%,65%)`) + Amber
- **Glassmorphism**: `backdrop-filter: blur` cards with translucent surfaces
- **Typography**: `Outfit` (display/headings) + `Inter` (body)
- **Animations**: Waveform bars, vinyl spin, orb drift, pulse glow, slide-in toasts
- **Buttons**: Gradient violet→rose with spring-physics hover scale
